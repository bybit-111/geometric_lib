# План тестирования

## 1. Цели и задачи тестирования

Целью тестирования является проверка корректности реализации функций вычисления площади и периметра для четырёх геометрических фигур.

**Задачи:**
- Обеспечить автоматизированную и воспроизводимую проверку логики вычислений с помощью unit-тестов.

## 2. Описание тестируемого продукта

**Обзор функциональности:**  
- Продукт представляет собой реализацию функций вычисления площади и периметра для четырёх геометрических фигур.

**Особенности и требования к продукту, которые должны быть протестированы:**
- Корректное вычисление площади и периметра всех фигур.  
- Попытка выявить недостатки в вычислениях.

## 3. Область тестирования

**Тестированию подлежат:**
- `circle.py`: `area(r)`, `perimeter(r)`  
- `rectangle.py`: `area(a, b)`, `perimeter(a, b)`  
- `square.py`: `area(a)`, `perimeter(a)`  
- `triangle.py`: `area(a, h)`, `perimeter(a, b, c)`

**Проверяются:**
- Корректность вычислений на положительных значениях;  
- Обработка нулевых входных данных.

## 4. Стратегия тестирования

**Общий подход, методы, техники и типы тестирования:**  
Тестирование выполняется с использованием фреймворка `unittest`.

Для каждой функции разработаны тестовые случаи, охватывающие:
- Граничные значения;  
- Положительные значения.

## 5. Критерии приемки

**Продукт считается успешно протестированным, если:**
- Все тесты проходят без ошибок;  
- Репозиторий публично размещён на GitHub.

## 6. Ожидаемые результаты

- Успешное выполнение всех unit-тестов.
